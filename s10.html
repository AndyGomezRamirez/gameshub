Estructura del Proyecto Angular 19+

Arquitectura y Organización del Proyecto

Para el examen, es fundamental comprender la estructura de un proyecto Angular moderno. El proyecto "PerfilFutbolista" sigue las mejores prácticas de organización que deberás aplicar.

Estructura de Carpetas Clave
Carpetas principales:
  • /src/app/ - Código principal de la aplicación
  • /src/app/inicio/ - Componente de página inicial
  • /src/app/jugadores/ - Módulo de funcionalidad
  • /src/app/services/ - Servicios de datos
Archivos de configuración:
  • app.config.ts - Configuración principal
  • app.routes.ts - Rutas principales
  • main.ts - Bootstrap de la aplicación
  • NO app.module.ts - ¡Componentes standalone!
Concepto Clave para el Examen: Componentes Standalone

En Angular 19+, los componentes standalone revolucionan la arquitectura:

  • Sin NgModules: Los componentes declaran sus propias dependencias en la propiedad imports
  • Configuración simplificada: Usa app.config.ts en lugar de app.module.ts
  • Mejor tree-shaking: Solo se incluye el código realmente utilizado
  • Comando de generación: ng generate component nombre --standalone
Puntos críticos para el examen:
  • NO crear archivos .module.ts para componentes standalone
  • Importar dependencias directamente en cada componente
  • Usar provideRouter(routes) en lugar de RouterModule.forRoot()
  • Entender la diferencia entre carga directa y lazy loading
Competencia esperada: Ser capaz de identificar y crear la estructura correcta de un proyecto Angular 19+ sin NgModules, configurar el bootstrapping con bootstrapApplication y organizar componentes de forma modular.

Enrutamiento y Lazy Loading

El enrutamiento es fundamental en Angular. Para el examen, debes dominar tanto la configuración básica como el lazy loading para optimización.

Estructura de Enrutamiento
1. Archivo principal de rutas (app.routes.ts):
  • Define rutas principales de la aplicación
  • Configura lazy loading con loadChildren
  • Exporta constante routes: Routes
2. Rutas de módulos específicos (jugadores.routes.ts):
  • Define rutas secundarias del módulo
  • Usa carga directa con component
  • Maneja parámetros de ruta con :id
Patrón de Lazy Loading en Angular 19+

El lazy loading se implementa de manera diferente con componentes standalone:

// En app.routes.ts
{
  path: 'jugadores',
  loadChildren: () => import('./jugadores/jugadores.routes')
    .then(m => m.JUGADORES_ROUTES)
}

Nota: Se cargan archivos de rutas, NO módulos NgModule

Errores comunes en el examen:
  • Intentar cargar NgModules en lugar de archivos de rutas
  • Olvidar exportar la constante de rutas
  • No usar provideRouter(routes) en app.config.ts
  • Confundir rutas absolutas con rutas relativas
Competencia esperada: Configurar enrutamiento complejo con lazy loading, manejar parámetros de ruta, y optimizar la carga de la aplicación mediante división de código.

Componentes Standalone y Servicios

Para el examen, debes dominar tanto la creación de componentes standalone como la implementación de servicios. Estos son los pilares fundamentales de cualquier aplicación Angular moderna.

Anatomía de un Componente Standalone
Características clave:
  • standalone: true en el decorador
  • Propiedad imports: [] para dependencias
  • No requiere declaración en NgModule
  • Gestiona sus propias dependencias
Imports comunes necesarios:
  • CommonModule - Para *ngIf, *ngFor
  • FormsModule - Para ngModel
  • RouterLink - Para navegación
  • RouterOutlet - Para componente principal
Servicios e Inyección de Dependencias

Los servicios en Angular siguen el patrón Singleton:

  • @Injectable({ providedIn: 'root' }) - Disponible en toda la aplicación
  • Se inyectan en el constructor de componentes
  • Usan RxJS Observables para datos asíncronos
  • Encapsulan lógica de negocio y acceso a datos

Patrón típico: getJugadores(): Observable

Errores frecuentes en el examen:
  • Olvidar standalone: true en componentes nuevos
  • No importar CommonModule al usar directivas estructurales
  • Intentar declarar componentes standalone en módulos
  • No suscribirse a Observables en servicios
  • Confundir providedIn: 'root' con providers locales
Competencia esperada: Crear componentes standalone con sus imports correctos, implementar servicios con inyección de dependencias, y manejar comunicación entre componentes a través de servicios utilizando RxJS Observables.

Data Binding y Directivas Estructurales

En el examen necesitarás demostrar dominio completo de los tipos de data binding y el uso efectivo de directivas estructurales para controlar el DOM dinámicamente.

Tipos de Data Binding Esenciales
Binding unidireccional:
  • {{ expresion }} - Interpolación
  • [propiedad]="valor" - Property binding
  • (evento)="método()" - Event binding
Binding bidireccional:
  • [(ngModel)]="propiedad" - Two-way binding
  • Requiere FormsModule importado
  • Ideal para formularios e inputs
Directivas Estructurales Críticas

Estas directivas modifican la estructura del DOM:

  • *ngIf="condicion" - Renderiza condicionalmente
  • *ngFor="let item of items" - Itera sobre arrays
  • *ngIf="datos; else templateVacio" - Con template alternativo
  • [ngClass]="{ 'clase': condicion }" - Clases CSS dinámicas
  • [ngStyle]="{ 'color': variable }" - Estilos dinámicos
Trampas comunes en el examen:
  • Usar *ngIf y *ngFor en el mismo elemento (prohibido)
  • Olvidar importar CommonModule para directivas estructurales
  • No manejar arrays vacíos o datos undefined
  • Confundir property binding [] con interpolación {{}}
Competencia esperada: Implementar interfaces dinámicas usando todos los tipos de data binding, manejar listas con *ngFor, condiciones con *ngIf, y aplicar estilos/clases CSS de forma reactiva.

RxJS y Manejo de Datos Asíncronos

Los Observables son fundamentales en Angular. Para el examen, debes entender cómo crearlos, suscribirte a ellos, y manejar datos asíncronos correctamente.

Patrones Esenciales de RxJS
En servicios:
  • of(data) - Crear Observable simple
  • from(array) - De array a Observable
  • map(), filter() - Transformar datos
  • catchError() - Manejo de errores
En componentes:
  • .subscribe() - Suscribirse a datos
  • async pipe - Suscripción automática
  • unsubscribe() - Evitar memory leaks
  • Manejo de estados de carga
Patrón Típico en Servicios Angular

Estructura estándar que debes dominar:

getJugadores(): Observable<Jugador[]> {
  return of(this.jugadores);
}

getJugadorPorId(id: number): Observable<Jugador | undefined> {
  const jugador = this.jugadores.find(j => j.id === id);
  return of(jugador);
}
Errores críticos con Observables:
  • Devolver datos directos en lugar de Observables
  • No suscribirse a Observables en componentes
  • Olvidar manejar errores con catchError
  • No usar el async pipe para simplificar suscripciones
  • Memory leaks por no hacer unsubscribe
Competencia esperada: Crear y consumir Observables correctamente, implementar patrones asíncronos robustos, y manejar estados de carga y error en interfaces de usuario.
